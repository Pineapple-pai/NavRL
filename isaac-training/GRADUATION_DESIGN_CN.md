# 基于强化学习的四旋翼无人机动态障碍避障导航研究（毕业设计版）

## 一、课题名称建议

可选题目如下：

基于强化学习的四旋翼无人机动态障碍环境自主导航方法研究

### 题目版本 2

面向动态障碍场景的四旋翼无人机强化学习避障导航系统设计与实现

基于 Isaac Sim 的四旋翼无人机动态避障强化学习训练与评估方法研究

## 二、研究背景与意义

随着无人机在巡检、物流、安防与复杂环境探测等任务中的应用增多，其在复杂障碍环境中的自主导航能力变得越来越重要。传统路径规划方法在静态环境中具有较强可解释性，但在动态障碍、多扰动和连续控制耦合场景中，往往难以同时兼顾感知变化、避障决策与稳定控制。

强化学习为无人机在复杂环境中的自主导航提供了一种端到端或近端到端的策略学习思路。通过在高保真仿真平台中构建静态与动态障碍环境，可以较低成本地完成大量训练与验证，从而为复杂场景下的导航策略研究提供支撑。

本课题基于 NavRL 与 Isaac Sim 4.x 仿真平台，围绕四旋翼无人机在静态与动态混合障碍环境中的自主避障导航问题，开展环境设计、奖励塑形、训练流程优化与评估展示工作，具有较强的工程实践意义和一定的研究展示价值。

## 三、研究目标

本课题拟实现以下目标：

- 搭建面向四旋翼无人机动态障碍避障任务的强化学习训练环境
- 优化环境重置逻辑，使任务目标更符合“真实穿越障碍区域”的要求
- 通过奖励函数修正与课程学习设计，提高策略在动态障碍场景中的稳定性与泛化能力
- 建立定性视频回放与定量统计评估相结合的结果分析体系
- 对比不同障碍配置与训练策略下的实验结果，总结影响导航性能的关键因素

### 1. 拟解决的核心问题

- 如何将四旋翼无人机动态障碍导航任务定义得更贴近“真实穿越障碍区并稳定到达目标区域”的目标，而不是停留在宽松的展示性通过判据上
- 如何通过环境重置逻辑、动态障碍约束和奖励塑形，缓解策略在动态扰动后的后段失稳与碰撞问题
- 如何构建一套既能支撑工程展示、又能支撑毕业设计结论的评估体系，使展示指标与正式统计指标相互区分
- 如何在现有 NavRL / OmniDrones 框架基础上，形成可复现、可比较、可分析的实验流程

### 2. 预期贡献概括

若按毕业设计成果表述，本课题的预期贡献可概括为：

- 面向四旋翼动态障碍导航任务，对仿真环境与任务定义进行针对性优化
- 结合奖励塑形、课程学习和失败模式分析，改进训练流程稳定性
- 提出展示指标与正式评估指标相区分的结果表达方式，提升实验结论严谨性
- 建立视频回放、批量统计、结果文档整理相结合的工程化实验闭环

### 3. 可作为毕业设计工作量的组成部分

从毕业设计工作量角度看，本课题已覆盖以下典型内容：

- 任务建模与场景配置

## 四、系统与方法概述

### 1. 仿真平台与基础框架

- 仿真平台：Isaac Sim 4.x
- 强化学习基础框架：NavRL / OmniDrones
- 核心实现语言：Python
- 学习与控制相关库：PyTorch、TorchRL
- 展示与数据处理：OpenCV、Shell、Markdown

### 2. 研究对象与任务场景

研究对象为四旋翼无人机，其任务是在静态障碍、动态障碍及混合障碍环境中，从起点出发完成避障与导航飞行。

当前重点场景包括：

- 纯静态障碍场景：`120` 个静态障碍
- 中等动态密度场景：`120 + 16`
- 动态压力测试场景：`100 + 32`
- 高密度混合场景：`120 + 32`

### 3. 当前已完成的核心改动

#### 3.1 环境与任务定义改动

- 修改环境重置逻辑，使无人机必须通过障碍区，而不是停留在过于简单的可行起始状态
- 对动态障碍生成方式增加约束，保护起点和终点附近关键通道
- 梳理多种障碍配置，形成可复现实验场景

#### 3.2 奖励函数与训练流程优化

- 调整奖励函数，增强避障之后的姿态稳定性约束
- 针对失败案例进行回放分析，识别后段失稳、动态扰动引起的间接碰撞等问题
- 结合课程学习思路，逐步提高场景难度

#### 3.3 评估与展示工具链建设

- 编写批量视频导出脚本，自动生成 `mp4` 视频与 `png` 截图
- 编写批量统计脚本，输出 `csv/json/md` 三类结果文件
- 在 README 中整合视频、截图和统计文件，形成项目展示材料

### 4. 方法特点概括

与单纯“训练一个策略模型”的项目相比，本课题的方法特点在于：

- 强调任务定义、环境分布与成功判据的一致性
- 强调通过失败回放反向修正环境逻辑与奖励设计
- 强调展示指标与正式统计指标的分层表达
- 强调可复现实验脚本、结果文件与答辩材料之间的统一组织

## 七、建议的实验设计

### 0. 实验组织原则

建议将毕业设计实验按“单变量优先、逐步增加复杂度”的方式组织，避免在同一次对比中同时改变过多因素。具体建议如下：

- 每组对比实验尽量只改变一个核心变量，如奖励函数、环境重置逻辑或动态障碍密度
- 训练超参数、checkpoint 选择规则、评估 seed 数量、并行环境数尽量保持一致
- 展示视频仅用于定性说明，不直接替代正式统计结论
- 正式结果优先引用批量评估文件中的 `evaluation success`、`collision rate`、`truncated rate` 等指标

### 1. 场景维度实验

建议从以下维度组织实验：

- 纯静态场景：`50`、`120`、`160`、`200` 个静态障碍
- 中等动态场景：`120 + 16`
- 动态压力场景：`100 + 32`
- 高密度混合场景：`120 + 32`

### 2. 方法维度实验

可进行以下对比：

- 原始环境逻辑 vs 修改后环境逻辑
- 原始奖励函数 vs 增强姿态稳定性奖励
- 无课程学习 vs 有课程学习
- 不同动态障碍分布约束设置对比

### 3. 评价指标建议

建议至少使用以下指标：

- 正式成功率
- 碰撞率
- 截断率
- 平均回报
- 平均回合长度
- 轨迹平滑性或姿态稳定性指标

其中建议明确区分以下概念：

- `showcase pass`
  - 用于视频与 PNG 展示
  - 更适合回答“这一批回放看起来是否顺利通过”
- `evaluation success`
  - 用于正式实验统计
  - 当前建议定义为：第一回合内连续 `k` 步进入目标区，且回合最终不是 `terminated`
  - 建议在论文中固定 `k=5` 或 `k=10`，并全文统一
- `goal contact rate`
  - 用于辅助说明策略是否曾接近目标，但不应单独视为正式成功率

### 4. 建议的变量控制方式

为了让实验更接近论文/毕设标准，建议固定以下设置：

- 相同的训练步数或相同的 checkpoint 选择规则
- 相同的评估脚本与正式 success 定义
- 相同的并行环境数，例如 `20` 个环境并行评估
- 相同的 seed 集合，例如先用 `5` 个 seed，最终结果补到 `10` 个 seed
- 相同的场景参数范围，仅在目标变量上做改动

### 5. 建议输出的标准化结果

建议最终论文或答辩中至少包含以下标准化结果：

- 总表：各场景下的 `evaluation success / collision / truncated / return / episode_len`
- 对比表：修改前后环境逻辑、奖励函数、课程学习设置的结果对比
- 图表：正式成功率与碰撞率柱状图
- 图表：典型失败模式分类图
- 附录：关键场景的视频链接与结果文件路径

### 6. 建议最终形成的毕设交付物

建议最终毕业设计至少形成以下交付物：

- 毕业论文正文
- 开题报告或任务书相关材料
- 中期检查材料
- 答辩 PPT
- 关键实验结果表与图
- 典型场景回放视频与结果摘要文件

### 答辩时建议重点展开的三条主线

- 第一条主线：任务定义是否合理
  - 说明为什么单纯的 `reach_goal` 或展示视频不足以支撑正式结论
  - 强调你对正式 success metric 的重新定义与指标分层
- 第二条主线：方法改进是否有针对性
  - 说明你的优化不是笼统调参，而是围绕环境逻辑、奖励塑形和失败模式做定向修正
- 第三条主线：结果是否可信
  - 说明结果来自多场景、多 seed 的批量评估，而不是单次回放现象
